# NPM Audit - 2026-05-09

## Contexto

Depois do ciclo de Dependabot, `npm audit` reportava 7 vulnerabilidades existentes. Esta revisão trata a auditoria separadamente dos PRs rotineiros de atualização para evitar misturar correções seguras de lockfile com mudanças de ecossistema.

## Correção aplicada

Foi aplicada uma atualização segura apenas no `package-lock.json`, dentro dos ranges já permitidos pelo projeto:

| Pacote | Origem | Antes | Depois | Resultado |
| --- | --- | --- | --- | --- |
| `express-rate-limit` | transiente via `shadcn` -> `@modelcontextprotocol/sdk` | `8.4.1` | `8.5.1` | Remove caminho vulnerável de `ip-address` |
| `ip-address` | transiente via `express-rate-limit` | `10.1.0` | `10.2.0` | Corrige advisory moderado |
| `fast-uri` | transiente via `ajv` | `3.1.0` | `3.1.2` | Corrige advisories altos |
| `hono` | transiente via `@modelcontextprotocol/sdk` | `4.12.15` | `4.12.18` | Corrige advisories moderados |

Não houve alteração em `package.json`.

## Resultado atual

Depois da correção segura, `npm audit` caiu de 7 para 3 vulnerabilidades:

| Pacote | Severidade | Direto | Motivo de pendência |
| --- | --- | --- | --- |
| `next` | moderada | sim | `next@16.2.6` ainda é a versão `latest`; o fix sugerido pelo audit é downgrade inseguro para `next@9.3.3` |
| `postcss` | moderada | não | Copiado dentro de `next`; depende de correção upstream do Next atual |
| `xlsx` | alta | sim | `xlsx@0.18.5` ainda é a versão `latest` publicada no npm e o audit não oferece `fixAvailable` |

## Decisões

- `npm audit fix --force` não foi usado.
- O downgrade sugerido para `next@9.3.3` foi rejeitado por ser uma regressão de framework e incompatível com o app atual.
- `xlsx` ficou pendente porque não há versão npm corrigida disponível no range atual.
- A substituição futura de `xlsx` deve ser tratada em PR dedicado, com validação de importação/exportação XLSX e SINAPI.

## Validação

Comandos executados nesta auditoria:

```bash
npm ci
npm audit
npm ls express-rate-limit fast-uri hono ip-address next postcss xlsx
```

Validação completa do PR:

```bash
npm run lint
npm run test
npm run build
```

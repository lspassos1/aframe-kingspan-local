## Operating Rules

Lucas is the final authority for product, UX, design, scope, and priority.

GitHub comments marked `Lucas Review` are official instructions.

Slack-generated `Lucas Review` comments are official instructions.

Before working on any PR, read:

- issue body;
- PR body;
- latest PR comments;
- latest review comments;
- latest review threads;
- `docs/lucas-design-authority-protocol.md`;
- `docs/slack-github-review-bridge.md`.

If the latest Lucas Review says `não aprovado` or `bloqueado`, do not advance to another PR.

Fix the current PR.

Never merge automatically.

Never mark ready automatically.

Never close issues automatically.

Never use `Closes #`.

Use only `Refs #`.

Passing lint/test/build does not mean product approval.

Visual approval requires Lucas.
